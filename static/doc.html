Evo Chatterbox Docs

System Overview

  • Local text-to-speech application with a web interface and FastAPI backend.
  • English uses the standard backend. All other supported languages use the multilingual backend.
  • Optional reference audio can be uploaded to approximate a target voice timbre.
  • CUDA is used automatically when an NVIDIA GPU is available; otherwise the system falls back to CPU.
  • Generated audio is returned as a WAV file.

Quality Modes

IDLabelBehavior
ultraHigh stabilitySplits text into smaller chunks with longer pauses. Best for non-English languages and long text.
maxBalanced stabilityDefault mode. Splits text into chunks for a balance between stability and naturalness.
fastDirect processingGenerates the full text in one pass. Fastest, but less stable on longer text.

API Endpoints

EndpointMethodDescription
/GETReturns the main web interface.
/healthGETReturns {"status":"ok"}.
/configGETReturns runtime info, supported languages, quality modes, and upload limit.
/generatePOSTGenerates a WAV file from input text and parameters.

GET /config response includes

device device_mode cuda_available gpu_name cpu_name default_language languages quality_modes audio_prompt_max_mb

POST /generate form fields

FieldTypeRequiredDefaultDescription
textstringYesNoneInput text to synthesize. Empty text is rejected.
language_idstringNoenLanguage code. Must match a supported language. Portuguese aliases pt-BR and pt-PT are accepted and map to the multilingual pt model.
quality_modestringNomaxOne of ultra, max, fast.
exaggerationfloatNo0.5Range 0.0 to 1.0.
cfg_weightfloatNo0.5Range 0.0 to 1.0.
temperaturefloatNo0.8Range 0.1 to 2.0.
audio_promptfileNoNoneOptional voice reference file. Accepted types: .wav, .mp3, .flac, .m4a, .ogg. Max size: 15 MB.

Response headers returned by /generate

HeaderDescription
X-Character-CountNumber of input characters processed.
X-Language-IdRequested language code used by the client, including aliases such as pt-BR or pt-PT.
X-Model-Language-IdActual model language code used internally, such as pt.
X-Backendstandard or multilingual.
X-Devicecpu or cuda.
X-Quality-ModeApplied quality mode.
X-Output-FilenameGenerated WAV file name.
X-Request-IdUnique request identifier for logs and tracing.

API Examples

cURL

curl -X POST "http://127.0.0.1:8000/generate" \
  -F "text=Hello, this is a voice test." \
  -F "language_id=en" \
  -F "quality_mode=max" \
  -F "exaggeration=0.5" \
  -F "cfg_weight=0.5" \
  -F "temperature=0.8" \
  --output chatterbox.wav

With reference audio

curl -X POST "http://127.0.0.1:8000/generate" \
  -F "text=Clone this voice." \
  -F "language_id=en" \
  -F "quality_mode=max" \
  -F "audio_prompt=@reference.wav" \
  --output chatterbox.wav

PowerShell and Python

PowerShell

$form = @{
  text = "Hello, this is a voice test."
  language_id = "en"
  quality_mode = "max"
  exaggeration = "0.5"
  cfg_weight = "0.5"
  temperature = "0.8"
}

Invoke-RestMethod -Uri "http://127.0.0.1:8000/generate" -Method Post -Form $form -OutFile "chatterbox.wav"

Python

import requests

response = requests.post(
    "http://127.0.0.1:8000/generate",
    data={
        "text": "Hello, this is a voice test.",
        "language_id": "en",
        "quality_mode": "max",
        "exaggeration": "0.5",
        "cfg_weight": "0.5",
        "temperature": "0.8",
    },
)

with open("chatterbox.wav", "wb") as f:
    f.write(response.content)

print(response.headers.get("X-Backend"))
print(response.headers.get("X-Device"))

How To Use The Web App

  1. Open http://127.0.0.1:8000/.
  2. Choose a language.
  3. Choose an emotion preset or leave the default.
  4. Select a quality mode.
  5. Optionally upload a reference audio file.
  6. Enter the text to synthesize.
  7. Click Generate audio.
  8. Listen in the built-in player or download the WAV file.

Operational Notes

  • The first model load can take longer than later requests.
  • English is typically faster than multilingual generation.
  • Long inputs are more stable in ultra or max mode.
  • Each request generates a temporary output file and returns it as a download.
  • Device selection can be influenced with the CHATTERBOX_DEVICE environment variable using auto, cpu, or cuda.
